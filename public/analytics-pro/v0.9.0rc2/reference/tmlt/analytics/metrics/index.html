class QuantileAbsoluteError(output, measure_column, quantile, join_columns, *, name=None, description=None, baselines=None)#

Bases: tmlt.analytics.metrics._base.JoinedOutputMetric

Computes the quantile of the absolute error.

Note

This class is only available on a paid version of Tumult Analytics. If you would like to hear more, please contact us at info@tmlt.io.

Note

This error metric is not analytic, and is calculated from a single run of the DP and baseline.

How it works: 1. Take the intersection of the groups in the DP and baseline. 2. Compute the absoulte error for each group. The absolute error is abs(dp - baseline). 3. Return the n-th quantile of the absolute error.

Example

>>> # Create the metric
>>> quantile_ae_X = QuantileAbsoluteError(
...     output="books_by_zip_code",
...     measure_column="X",
...     quantile=0.5,
...     join_columns=["A"]
... )
>>> quantile_ae_X.quantile
0.5
>>> quantile_ae_X.join_columns
['A']
>>> quantile_ae_X.compute_for_baseline(dp_outputs, baseline_outputs)
10.0

class JoinedOutputMetric(output, join_columns, name, description, baselines)#

Bases: SingleBaselineMetric, abc.ABC

Base class for metrics computing value from combined DP and single baseline.

Note

This class is only available on a paid version of Tumult Analytics. If you would like to hear more, please contact us at info@tmlt.io.

Subclasses of JoinedOutputMetric define a compute_on_joined_output method which takes in a single dataframe, the result of joining the DP and baseline output tables with the given name on the given list of columns, and returns the metric value. The joined table is the result of performing an outer join between the DP and baseline tables on the given join columns.

This class is only available on a paid version of Tumult Analytics. If you would like to hear more, please contact us at info@tmlt.io.

Computes the quantile of the relative error.

Note

This class is only available on a paid version of Tumult Analytics. If you would like to hear more, please contact us at info@tmlt.io.

Note

This error metric is not analytic, and is calculated from a single run of the DP and baseline.

How it works: 1. Take the intersection of the groups in the DP and baseline. 2. Compute the relative error for each group. The relative error is abs(dp - baseline) / baseline. 3. Return the n-th quantile of the relative error.

class SpuriousRate(output, join_columns, *, name=None, description=None, baselines=None)#

Bases: tmlt.analytics.metrics._base.SingleBaselineMetric

Computes the fraction of groups in the DP output that aren’t in the baseline.

Note

This class is only available on a paid version of Tumult Analytics. If you would like to hear more, please contact us at info@tmlt.io.

Note

This error metric is not analytic, and is calculated from a single run of the dp and baseline.

Note

below, released means that the group is in the DP output, and spurious means that the group is not in the baseline output.

How it works: 1. Take the left anti join of the groups in the DP and baseline. This returns all rows from the DP where there is no match in the baseline i.e., spurious released count. 2. Compute the spurious rate for each group. The spurious rate is spurious released count / released count.

...     join_columns=["A"]
... )

class SuppressionRate(output, join_columns, *, name=None, description=None, baselines=None)#

Bases: tmlt.analytics.metrics._base.SingleBaselineMetric

Computes the fraction of groups in the baseline that aren’t in the output.

Note

This class is only available on a paid version of Tumult Analytics. If you would like to hear more, please contact us at info@tmlt.io.

Note

This error metric is not analytic, and is calculated from a single run of the component and baseline.

Note

below, released means that the group is in the DP output, and spurious means that the group is not in the output of the baseline.

How it works: 1. Take the left anti join of the groups in the baseline and DP. This returns all rows from the baseline where there is no match in the DP output i.e., non-spurious non-released count 2. Compute the suppression rate for each group. The suppression rate is non-spurious non-released count / non-spurious count.

Example

>>> # Create the metric
>>> suppression_rate = SuppressionRate(
...     output="books_by_zip_code",
...     join_columns=["A"]
... )
>>> suppression_rate.join_columns
['A']
>>> suppression_rate.compute_for_baseline(dp_outputs, baseline_outputs)
0.25
