.. py:class:: QuantileRelativeError(output, measure_column, quantile, join_columns, *, name = None, description = None, baselines = None)

   Bases: :class:`tmlt.analytics.metrics._base.JoinedOutputMetric`

   Computes the quantile of the relative error.

   .. note::

       This class is only available on a paid version of Tumult Analytics. If you
       would like to hear more, please contact us at info@tmlt.io.

   .. note::
       This error metric is not analytic, and is calculated from a single run of the
       DP and baseline.

   How it works:
   1. Take the intersection of the groups in the DP and baseline.
   2. Compute the relative error for each group. The relative error is
   abs(dp - baseline) / baseline.
   3. Return the n-th quantile of the relative error.


   .. rubric:: Example

   ..
       >>> import pandas as pd
       >>> from pyspark.sql import SparkSession
       >>> spark = SparkSession.builder.getOrCreate()
       >>> dp_df = spark.createDataFrame(
       ...     pd.DataFrame(
       ...         {
       ...             "A": ["a1", "a1", "a2"],
       ...             "X": [50, 110, 100]
       ...         }
       ...     )
       ... )
       >>> dp_outputs = {"books_by_zip_code": dp_df}
       >>> baseline_df = spark.createDataFrame(
       ...     pd.DataFrame(
       ...         {
       ...             "A": ["a1", "a1", "a2"],
       ...             "X": [100, 100, 100]
       ...         }
       ...     )
       ... )
       >>> baseline_outputs = {"books_by_zip_code": baseline_df}

   >>> # Create the metric
   >>> quantile_re_X = QuantileRelativeError(
   ...     output="books_by_zip_code",
   ...     measure_column="X",
   ...     quantile=0.5,
   ...     join_columns=["A"]
   ... )
   >>> quantile_re_X.quantile
   0.5
   >>> quantile_re_X.join_columns
   ['A']
   >>> quantile_re_X.compute_for_baseline(dp_outputs, baseline_outputs)
   0.1

   Computes the fraction of groups with relative error above a threshold.

   .. note::

       This class is only available on a paid version of Tumult Analytics. If you
       would like to hear more, please contact us at info@tmlt.io.

   .. note::
       This error metric is not analytic, and is calculated from a single run of the
       component and baseline.

   How it works:
   1. Take the intersection of the groups in the component and baseline.
   2. Compute the relative error for each group.
   3. Return the fraction of groups with relative error above the threshold.


   .. rubric:: Example

   ..
       >>> import pandas as pd
       >>> from pyspark.sql import SparkSession
       >>> spark = SparkSession.builder.getOrCreate()
       >>> dp_df = spark.createDataFrame(
       ...     pd.DataFrame(
       ...         {
       ...             "A": ["a1", "a1", "a2"],
       ...             "X": [50, 110, 100]
       ...         }
       ...     )
       ... )
       >>> dp_outputs = {"books_by_zip_code": dp_df}
       >>> baseline_df = spark.createDataFrame(
       ...     pd.DataFrame(
       ...         {
       ...             "A": ["a1", "a1", "a2"],
       ...             "X": [100, 100, 100]
       ...         }
       ...     )
       ... )
       >>> baseline_outputs = {"books_by_zip_code": baseline_df}

   >>> # Create the metric
   >>> high_re_fraction_X = HighRelativeErrorFraction(
   ...     output="books_by_zip_code",
   ...     measure_column="X",
   ...     relative_error_threshold=0.25,
   ...     join_columns=["A"]
   ... )
   >>> high_re_fraction_X.relative_error_threshold
   0.25
   >>> high_re_fraction_X.join_columns
   ['A']
   >>> high_re_fraction_X.compute_for_baseline(dp_outputs, baseline_outputs)
   0.4
