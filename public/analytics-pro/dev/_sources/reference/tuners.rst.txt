.. note::

Parameter tuning and optimization in Tumult is done with the
:class:`~tmlt.tune.SessionProgramTuner` class, an abstract base class that
defines the interface for tuning :class:`~tmlt.analytics.SessionProgram`\ s.

To tune a specific program, users should subclass
:class:`~tmlt.tune.SessionProgramTuner`, passing their
:class:`~tmlt.analytics.SessionProgram` as the ``program`` class argument.

.. ::
    >>> import math
    >>> from tmlt.analytics import (
    ...     AddOneRow,
    ...     KeySet,
    ...     PureDPBudget,
    ...     QueryBuilder,
    ...     SessionProgram,
    ...     Session,
    ... )
    >>> from tmlt.tune import (
    ...     metric,
    ...     MedianRelativeError,
    ...     SessionProgramTuner,
    ...     Tunable,
    ... )
    >>> from pyspark.sql import DataFrame, GroupedData
    >>> from pyspark.sql import SparkSession
    >>> import pyspark.sql.functions as sf
    >>> from typing import Dict
    >>> spark = SparkSession.builder.getOrCreate()

>>> class Program(SessionProgram):
...     class ProtectedInputs:
...         protected_df: DataFrame
...     class Outputs:
...         b_sum: DataFrame
...     class Parameters:
...         low: int
...         high: int
...     def session_interaction(self, session: Session):
...         low = self.parameters["low"]
...         high = self.parameters["high"]
...         a_values = KeySet.from_dict({"a": ["x", "y"]})
...         sum_query = QueryBuilder("protected_df").groupby(a_values).sum("b", low, high)
...         b_sum = session.evaluate(sum_query, self.privacy_budget)
...         return {"b_sum": b_sum}
>>> class Tuner(SessionProgramTuner, program=Program):
...     @joined_output_metric(name="root_mean_squared_error", output="b_sum", join_columns=["a"])
...     @staticmethod
...     def compute_rmse(joined_output: DataFrame):
...         err = sf.col("b_sum_dp") - sf.col("b_sum_baseline")
...         rmse = joined_output.agg(sf.sqrt(sf.avg(sf.pow(err, sf.lit(2)))).alias("rmse"))
...         return rmse.collect()[0]["rmse"]
...
...     metrics = [
...         MedianRelativeError(
...             output="b_sum",
...             measure_column="b_sum",
...             name=f"mre_{index}",
...             join_columns=["a"],
...         )
...     ]


Just like a :class:`~tmlt.analytics.SessionProgram`, once a subclass of
:class:`~tmlt.tune.SessionProgramTuner` is defined, it can be instantiated using the
automatically-generated builder for that class. Unlike a
:class:`~tmlt.analytics.SessionProgram`, you can pass :class:`~tmlt.tune.Tunable`
objects to the builder methods instead of concrete values.

>>> protected_df = spark.createDataFrame([("x", 2), ("y", 4)], ["a", "b"])
>>> tuner = (
...     Tuner.Builder()
...     .with_privacy_budget(Tunable("budget"))
...     .with_private_dataframe("protected_df", protected_df, AddOneRow())
...     .with_parameter("low", 0)
...     .with_parameter("high", Tunable("high"))
...     .build()
... )

The :meth:`~tmlt.tune.SessionProgramTuner.run` method can be used to run the
program to get the outputs of the DP and baseline programs.

>>> outputs = tuner.run({"budget": PureDPBudget(1), "high": 1})

The :meth:`~tmlt.tune.SessionProgramTuner.error_report` method on the tuner can be used to
run the program to get the DP and baseline outputs as well as the metrics defined in
the Tuner class.

>>> tuner.error_report({"budget": PureDPBudget(1), "high": 1}).show()  # doctest: +SKIP
Error report ran with budget PureDPBudget(epsilon=1) and the following tunable parameters:
budget: PureDPBudget(epsilon=1)
high: 1
and the following additional parameters:
low: 0
<BLANKLINE>
Metric results:
+---------+-------------------------+-------------------------------------------------------+
|   Value | Metric                  | Description                                           |
+=========+=========================+=======================================================+
|    0.5  | mre                     | Median relative error for column b_sum of table b_sum |
+---------+-------------------------+-------------------------------------------------------+
|    3.16 | root_mean_squared_error | User-defined metric (no description)                  |
+---------+-------------------------+-------------------------------------------------------+

Another illustrated example of how to use a :class:`~tmlt.tune.SessionProgramTuner`
to tune parameters can be found in the :ref:`Tuning parameters` tutorial.

----------------------------------

with the automatically-generated :class:`~tmlt.tune.SessionProgramTuner.Builder`. 
Each parameter can be specified with a fixed value or with :class:`~tmlt.tune.Tunable`,
to measure error for different values of this parameter.

NoPrivacySession
----------------
To compute baselines, the :class:`~tmlt.tune.SessionProgramTuner` relies on the
:class:`~tmlt.tune.NoPrivacySession`, a class with the same interface as a
:class:`~tmlt.analytics.Session`, but which can run queries without any privacy guarantees. 
Users should generally not use the :class:`~tmlt.tune.NoPrivacySession` directly.
