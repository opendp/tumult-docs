API reference#

The API of the Tumult Platform is split in three distinct components.

  • Tumult Analytics allows users to compute statistical queries on tabular data using differential privacy. The Tumult Analytics API also includes the fundamental abstractions used throughout the Tumult Platform to capture privacy guarantees.

  • Tumult Synthetics allows users to generate differentially private synthetic data.

  • Tumult Tune allows users to measure and optimize the accuracy of differentially private programs.

Tumult Analytics#

The Tumult Analytics API reference is split in three major sections.

  • Specifying privacy guarantees introduces the Session, the fundamental abstraction used to enforce differential privacy guarantees on sensitive data. It also lists classes used to define privacy budgets and protected changes.

  • Building queries introduces the classes and methods used to define statistical queries to evaluate with differential privacy.

  • Defining and using programs introduces the SessionProgram, a way to define structured differential privacy programs that can be used with higher-level tools, such as Tumult Tune.

Two further pages list configuration options and utility functions.

Tumult Synthetics#

The Tumult Synthetics API reference lists classes and methods to generate synthetic data.

Tumult Tune#

The Tumult Tune API reference is split in two sections.

  • Tuning programs introduces the SessionProgramTuner, the main class used to generate error reports. It also lists the various ways that users can interact with error reports.

  • Defining error metrics lists the built-in error metrics provided by Tumult Tune, and the methods and classes used to define custom error metrics.
